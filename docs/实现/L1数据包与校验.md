# L1 数据包与校验（实现状态）

本文档与 `微内核与Mod扩展.md` §9 对照，描述**当前仓库**已落地的 L1 行为与制包约定。

## 目录与发现

- 路径：`unity/Assets/StreamingAssets/Mods/<文件夹>/`
- 至少存在以下之一才会被识别为包：`mod.json`、`sim_params.json`、`wildlife.json`、`culture_skills.json`（文件名见 `L1DataFiles`）。
- 无 `mod.json` 时以**文件夹名**作为 `id`。

## 清单 `mod.json`

| 字段 | 说明 |
|------|------|
| `id` / `name` / `version` / `kind` | 与 §9.2 一致；`kind: core` 在同依赖层内优先于非 core。 |
| `loadPriority` | 整数，**同拓扑批次**内：值**小**的先合并，值**大**的后合并（后者覆盖前者）。默认 0。 |
| `dependencies` | 硬依赖：缺包或版本不满足时**整批不加载**并报错。 |
| `conflicts` | 与列出的 `id` 同批并存时报错。 |

## 依赖与合并顺序

1. 拓扑排序（有环则失败）。  
2. 同 `inDegree==0` 的批次内排序：`core` > `loadPriority` 升序 > `id` 字序。  
3. 各数据文件按此顺序**叠放**；同键以后写为准（`wildlife` / `culture_skills` 的 `clearBaseline` 会清空已合并表后再应用本文件）。

## 各数据文件

- `sim_params.json`：字段名为 `SimParams` **公开**字段名，未知 key 在控制台 **Warning**（不应用）。  
- `wildlife.json` / `culture_skills.json`：见各 `*FileV1` 与 `*TableBuilder` 注释；非法依赖图会拒绝加载。  
- `l1_initial_persistent.json`（可选）：新游戏开局读入，按本包 `id` 写入 L1 会话，并随 F5 进入存档的 `l1ModPersistent` 分桶。

## 存档

- 写入：`l1ModFolders`、`l1ModDisplayNames`、**`l1ModIds`**、`l1ModPersistent`。  
- 读档时：先 `L1ModLoader.Load()` 并 `L1ModSession.ApplyFrom`；**L1 加载失败则拒绝读档**。  
- 若存档含 **`l1ModIds` 且非空**：当前 L1 的 mod **id 顺序**必须与其**完全一致**（不区分大小写），否则**拒绝读档**并说明存/现差异。  
- **旧档**无 `l1ModIds`：不做强 id 校验，仅对文件夹顺序作与原先一致的**警告**（不阻止加载）。

## 未包含（非 L1）

- L2 沙盒、脚本、每 Tick 钩子等见微内核 **§4**。
