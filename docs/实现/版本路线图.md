## 里程碑一：纸面原型验证（Paper Prototype）

### 目标
> 用纸笔或电子表格模拟 MVP 场景（06 #18），验证核心体验假设。

### 具体任务
1. [ ] 在纸上画出极小地图（河流 + 上下游两块区域）
2. [ ] 为每只猿写一张卡片（名字/年龄/性格/位置）
3. [ ] 定义每回合的规则（觅食→social→移动→事件检查）
4. [ ] 手动推演 10 个回合（约等于 MVP 的 30 分钟游戏时间）
5. [ ] 在第 5 回合引入旱灾，第 7 回合提供降雨选项
6. [ ] 记录：推演中是否自然产生了"在乎某只猿"的感觉？

### 完成标准
- 能讲出至少 1 个个体的"故事"
- 旱灾选择让推演者感到真实的犹豫
- 如果两项都不成立 → 回去调整核心循环设计

### 预计耗时：1-2 天

## 里程碑二：最小技术原型（Tech Prototype）

### 目标
> 屏幕上有一群猿在动，它们会觅食、会饿、会死。玩家能看到它们。

### 具体任务
1. [ ] 选择技术栈（推荐：Godot 4 / Unity 2D / 纯 Python + Pygame — 哪个你最熟悉用哪个）
2. [ ] 实现 2D 网格地图渲染（地形色块即可，不需要美术）
3. [ ] 实现个体（Entity）数据结构
4. [ ] 实现基本需求系统（饥饿 + 觅食）
5. [ ] 实现移动系统（寻找食物 → 移动 → 吃 → 移动）
6. [ ] 实现死亡（饥饿值归零 → 死亡 → 从世界移除）
7. [ ] 实现时间控制（暂停/1x/2x/4x）
8. [ ] 点击个体显示信息（ID/年龄/饥饿度）

### 完成标准
- 20 只猿在地图上自主觅食
- 食物不够时有猿饿死
- 玩家能暂停、加速、点击查看
- **没有美术、没有 UI 框架、没有音效** — 纯功能

### 预计耗时：1-2 周

---

## 里程碑三：社交与命名（Social + Naming）

### 目标
> 猿之间有关系，玩家能给它们起名字。这是情感绑定的起点。

### 具体任务
1. [ ] 实现繁殖系统（配对 → 后代 → 亲子关系）
2. [ ] 实现简化性格（勇敢/谨慎 + 好奇/保守，随机生成，部分遗传）
3. [ ] 实现基本社交行为（接近、回避、简单的亲近检测）
4. [ ] 实现命名功能（点击个体 → 输入名字 → 名字显示在个体上方）
5. [ ] 实现个体视觉差异（大小、颜色深浅 — 能区分个体即可）
6. [ ] 实现生命周期（出生 → 成长 → 成年 → 衰老 → 死亡）
7. [ ] 实现简单事件日志（"[名字] 出生了" / "[名字] 死亡了"）

### 完成标准
- 玩家能追踪一只猿从出生到死亡的完整生命
- 看到它的后代出生时有一点"哦"的感觉
- 当它死的时候有一点"啊"的感觉
- 如果完全没感觉 → 个体差异性不够，回去加强

### 预计耗时：1-2 周

---

## 里程碑四：环境压力与选择（MVP 场景实现）

### 目标
> 完整实现 06 #18 定义的 MVP 场景。这是核心体验的第一次完整测试。

### 具体任务
1. [ ] 实现河流系统（水位变量 → 影响周围食物产出）
2. [ ] 实现旱灾事件（水位逐渐下降 → 食物减少 → 猿开始挨饿）
3. [ ] 实现干预系统（旱灾达到阈值 → UI 弹出"降雨"选项）
4. [ ] 实现降雨效果（上游水位恢复 → 下游水位下降）
5. [ ] 实现另一个猿的群体（下游，玩家初始不会注意到）
6. [ ] 实现简单的编年史（事件列表，可按时间浏览）
7. [ ] 让 3 个不同的人试玩 30 分钟

### 验证指标（来自 06 #18，四项中任意两项即可继续）
1. 玩家是否真的给个体命名了？
2. 玩家是否产生了情感（关心、担忧、紧张）？
3. 玩家是否愿意干预（或者犹豫过要不要干预）？
4. 当发现副作用时，玩家是否产生了内在冲突？

### 完成标准
- 3 个测试者中至少 2 个满足上述 4 项中的 2 项
- 如果不满足 → 分析原因，决定调整核心循环还是呈现方式

### 预计耗时：2-3 周

---

## 里程碑五：涌现行为萌芽（Emergence）

### 目标
> 猿开始做出"让你觉得它们在学什么"的行为。

### 具体任务
1. [ ] 实现完整性格系统（4 轴）
2. [ ] 实现记忆系统（地点记忆 + 个体记忆）
3. [ ] 实现简单情绪（平静/压力/哀伤）
4. [ ] 实现观察学习（幼崽观察附近成年猿的行为并有概率习得）
5. [ ] 实现第一个"工具行为"：捡起石头 → 敲击坚果（偶然触发 + 练习提升成功率）
6. [ ] 实现文化断裂检测（某技能的最后掌握者死亡 → 技能失传通知）
7. [ ] 实现其他动物物种（1-2 种：猎物 + 捕食者）

### 完成标准
- 经过足够时间后，有猿自主学会石头敲坚果
- 技能在群体中传播（至少传给一个后代）
- 群体分化开始出现（不同群体因不同经历产生不同行为模式）

### 预计耗时：3-4 周

---

## 里程碑后：方向判断点

完成里程碑五后，你应该有一个：
- 能运行的原型
- 至少 3 次外部试玩反馈
- 对"这个游戏是否好玩"的初步判断

此时做出关键决策：

| 如果… | 则… |
|-------|-----|
| 核心体验成立（人们在乎、人们犹豫） | 进入垂直切片阶段：美术风格、音效、UI、完整30分钟体验 |
| 个体有趣但循环不够 | 回到核心循环调整：增加事件密度、调整时间节奏 |
| 技术瓶颈（性能不行） | 转向探索 LOD / 简化模拟方案（见 06 #19, 08 §12） |
| 完全不好玩 | 重新审视核心概念：是"观察"本身不够，还是反馈不够？ |

## 时间线概览

```
2月下旬       3月上旬       3月           4月           5月
   │             │            │             │             │
   ▼             ▼            ▼             ▼             ▼
 纸面原型     技术原型     社交+命名     MVP场景     涌现行为
 (1-2天)     (1-2周)      (1-2周)      (2-3周)     (3-4周)
                                          ↑
                                     第一次外部测试
```

> **核心原则**：每个里程碑结束后都做评估。如果不通过就回头调，不要硬往前推。
> 独立开发的流程是：**设计 → 原型 → 推翻 → 再设计 → 再原型**。
