# 核心循环（Core Loop）

> 本文档回答一个问题：**玩家在每个时间尺度上到底在做什么？**
> 这是从概念到原型的桥梁。不涉及哲学论证，只谈可执行的行为与反馈。

---

## 0. 核心循环一句话总结

**观察 → 关注 → 投入情感 → 被迫选择 → 承受后果 → 重新观察**

玩家的"操作"极少，但"决策"极多——每一次移动视角都是决策，每一次命名都是承诺，每一次加速时间都是放弃。

---

## 1. 每分钟在做什么？（微循环 / Micro Loop）

### 玩家行为

| 行为 | 频率 | 本质 |
|------|------|------|
| 移动视角 / 切换观察尺度 | 高频 | 注意力分配（不可逆的情感投资） |
| 跟随某个个体 | 中频 | 建立连接（开始"在乎"某个生命） |
| 观察个体行为细节 | 持续 | 阅读世界（判断发生了什么、为什么） |
| 调整时间流速 | 中频 | 决定"跳过什么"（放弃某段历史） |
| 查看个体档案 / 信息层 | 低频 | 深入理解（确认猜测） |

### 系统反馈

- 个体正在执行日常行为：觅食、移动、社交互动（理毛、玩耍、威胁、安抚）。
- 环境在缓慢变化：天气、光线、季节信号。
- 偶发微事件：一只猿捡起石头、两个个体发生冲突、一只幼崽模仿母亲的动作。
- 世界在**玩家没看的地方**同样运转（没有任何提示）。

### 微循环的核心体验

> **"那只猿在干什么？"** — 好奇心驱动的持续观察。

玩家通过观察建立对个体和群体的理解。大部分时间是安静的、陪伴式的。偶尔出现的异常行为（一个新动作、一次冲突、一次受伤）让玩家保持注意力。

---

## 2. 每十分钟在做什么？（中循环 / Meso Loop）

### 玩家行为

| 行为 | 频率 | 本质 |
|------|------|------|
| 命名感兴趣的个体 | 低频 | 情感承诺（命名 = 开始在乎） |
| 标记关注事件 | 低频 | 设定自己的"叙事焦点" |
| 在群体/区域之间切换 | 中频 | 注意力的再分配 |
| 使用时间加速跳过平静期 | 中频 | 用时间换事件密度 |
| 查看关系网络 / 生态层 | 低频 | 理解结构性变化 |

### 系统反馈

- **环境事件**发生：季节变化带来食物丰枯、天气事件（暴风雨、干旱前兆）、捕食者出没。
- **社交事件**发生：个体间的地位挑战、配偶关系建立、幼崽出生、老年个体衰弱。
- **行为变化**可被察觉：某个个体开始更频繁地使用石头、某个群体的觅食范围在扩大。
- **通知**（如果玩家开启了）：被追踪个体的重大事件、新行为首次出现。

### 中循环的核心体验

> **"这个群体最近怎么了？"** — 模式识别驱动的理解与担忧。

玩家开始注意到趋势：食物在减少、群体在变大、某只被命名的猿开始有新行为。玩家从"看个体动作"上升到"理解群体状态"。10 分钟尺度是**情感绑定形成的关键窗口**——一旦你给一只猿起了名字并跟踪了 10 分钟，你开始在乎它的命运。

## 3. 每小时在做什么？（宏循环 / Macro Loop）

### 玩家行为

| 行为 | 频率 | 本质 |
|------|------|------|
| 经历代际更替 | 自然发生 | 见证生命的有限性 |
| 面对干预选择（若触发） | 极低频 | 核心困境：救谁？不救谁？ |
| 浏览世界编年史 | 中频 | 发现"你没看到的历史" |
| 探索新区域 | 低频 | 发现平行世界（另一个群体的命运） |
| 存档 / 回溯 | 低频 | 思想实验："如果那件事没发生呢？" |
| 比较不同群体的发展 | 中频 | 感受演化的随机性与趋同性 |

### 系统反馈

- **代际事件**：你命名的猿死了，它的后代继承了它的一些行为习惯。
- **文化里程碑**：某项技能在群体中开始扩散，或因最后传承者死亡而失传。
- **演化信号**：群体行为模式出现可辨识的变化——从偶然使用石头到有意识选择石头。
- **生态震荡**：种群数量波动、捕食者-猎物关系逆转、某物种区域性灭绝。
- **干预窗口**（若系统设计如此）：世界到达某种"可被回应"的状态，玩家获得一次选择机会。
- **事后发现**：翻阅编年史，发现一个你从未观察过的区域曾经有一个繁荣的族群——已经灭绝了。

### 宏循环的核心体验

> **"我见证了什么？我错过了什么？"** — 时间跨度产生的敬畏感与遗憾感。

这是情感收获的主要窗口。玩家在一小时内会经历：被命名个体的生老病死、群体的兴衰分合、自己注意力分配的后果（"要是我没切走就好了"），以及偶尔的干预困境。**每一个小时都应该产生至少一个"值得回味"的片段。**

## 4. 三层循环的嵌套关系

```
┌─────────────────────────────────────────────────────┐
│  宏循环（小时级）                                     │
│  见证 → 失去 → 发现 → 困境 → 承受                     │
│  ┌───────────────────────────────────────────────┐   │
│  │  中循环（十分钟级）                              │   │
│  │  关注 → 命名 → 追踪 → 理解趋势 → 担忧           │   │
│  │  ┌─────────────────────────────────────────┐   │   │
│  │  │  微循环（分钟级）                          │   │   │
│  │  │  观察 → 好奇 → 移动视角 → 发现细节          │   │   │
│  │  └─────────────────────────────────────────┘   │   │
│  └───────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

**驱动力传递**：
- 微循环的好奇心驱动玩家**持续观察**（不无聊）。
- 中循环的情感绑定驱动玩家**在乎结果**（不冷漠）。
- 宏循环的不可逆选择驱动玩家**反思与重玩**（不一次性）。

如果微循环失败（前几分钟就无聊了），后面两层循环不可能成立。
如果只有微循环成功（有趣但不在乎），游戏变成屏保。
三层必须同时运转。

## 5. 核心循环中的"变强"机制

本游戏没有传统意义上的"玩家变强"。取而代之的是：

### 世界在"变复杂"
- 猿类行为从本能走向有意识。
- 工具使用从偶然走向传统。
- 社会结构从松散走向复杂。
- 通讯从简单信号走向组合表达。

### 玩家在"变深入"
- 对世界的理解从陌生走向熟悉（知道去哪里看精彩的东西）。
- 对个体的情感从旁观走向绑定（开始在乎谁活谁死）。
- 对系统的认知从困惑走向洞察（能预判什么事件即将发生）。
- 对干预后果的理解从天真走向谨慎（知道"救"也可能是"杀"）。

### "下一轮循环"的驱动力

| 驱动力 | 来源 |
|--------|------|
| "这个群体接下来会怎样？" | 未完成叙事的吸引力 |
| "另一边发生了什么？" | 信息不对称的好奇心 |
| "如果当时我没干预呢？" | 回溯与分支的实验欲 |
| "换个种子会怎样？" | 不同初始条件的探索欲 |
| "这次我要关注狼群" | 多物种观察的新鲜感 |

## 6. 循环失败的风险与对策

### 风险一：微循环无聊（前几分钟就走了）

- **症状**：个体行为太重复、缺少可辨识的微事件、画面太静态。
- **对策**：
  - 确保个体行为有足够的视觉多样性（不同的觅食方式、社交动作、移动姿态）。
  - 确保环境有持续的微变化（光线、天气、声音景观）。
  - 当无事发生时，时间加速系统应足够智能地跳过空白期。
  - **原型验证重点**：盯着一只猿看 5 分钟，是否至少观察到 3 种不同的行为？

### 风险二：中循环无感（观察了但不在乎）

- **症状**：个体之间缺乏可辨识性、没有足够多的社交事件、群体没有可读的动态。
- **对策**：
  - 强化个体差异的视觉可辨识性（体型、伤疤、行为习惯的差异）。
  - 社交事件（冲突、理毛、幼崽出生、老年死亡）的密度要够。
  - 命名系统需要足够方便——命名的心理成本越低，情感绑定越快。
  - **原型验证重点**：观察一个群体 15 分钟后，玩家能否说出 "我最在乎的那只是___"？

### 风险三：宏循环空洞（玩了一小时但没有"时刻"）

- **症状**：一小时内没有任何值得回忆的事件、代际更替没有情感重量、无困境发生。
- **对策**：
  - 确保生态压力足够制造事件（资源不是均匀充裕的、捕食者存在威胁）。
  - 确保代际更替的呈现有可读性（不只是老的消失新的出现，而是有过渡期的衰老与交接）。
  - 干预机会不能太稀少——至少在一小时内给玩家一次"我是否要做点什么"的念头余波。
  - **原型验证重点**：玩了一小时后，玩家能否讲述 "在我的世界里发生了___"？

## 7. 与 MVP 原型场景的对应（06 #18）

MVP 场景（河流地图 / 两群猿类 / 旱灾 / 一次降雨干预）验证的正好是核心循环的三层：

| 循环层 | MVP 中如何验证 |
|--------|---------------|
| 微循环 | 玩家是否自然被个体行为吸引、是否观察到觅食与社交细节 |
| 中循环 | 玩家是否给个体命名、是否开始追踪特定个体或群体 |
| 宏循环 | 旱灾带来的干预选择、事后发现下游群体的信息不对称 |

MVP 的 30 分钟时长已足以覆盖微循环和中循环的验证；旱灾事件将宏循环压缩到可验证的范围内。

## 8. 核心循环的节奏曲线（目标）

```
情感强度
  ▲
  │          ╱╲        ╱╲
  │    ╱╲  ╱    ╲    ╱    ╲      ╱╲
  │  ╱    ╲╱      ╲  ╱      ╲  ╱    ╲
  │╱                ╲╱        ╲╱      ╲
  ┼───────────────────────────────────────▶ 时间
  0   5   10   15   20   25   30分钟

  安静  微惊喜  命名  追踪  担忧  事件  选择  后果
```

- **0-5 分钟**：安静的适应期。玩家认识世界、找到第一个感兴趣的个体。
- **5-10 分钟**：微事件吸引注意力。第一次命名可能发生在这里。
- **10-20 分钟**：情感绑定期。玩家开始关心被追踪个体的遭遇。
- **20-30 分钟**：事件高峰。环境压力显现，困境出现（如果使用 MVP 场景的旱灾设计）。
- **30 分钟+**：后果与发现。干预的后果展开，未观察区域的历史被发现。

> **这条曲线不是被编排的，而是通过初始条件的设计来提高其自然发生的概率。**
