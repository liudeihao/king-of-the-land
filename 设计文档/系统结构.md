# 系统结构（System Architecture）

> 本文档从"可实现"角度拆解游戏需要哪些系统、系统之间如何耦合。
> 不重复 02/03 的哲学论证，只关注系统边界、输入输出、优先级。

---

## 1. 世界生成系统

> 游戏开始前运行一次，生成初始世界。

### 输入
- 种子值（随机或玩家指定）
- 可选偏好（地理类型、气候倾向、物种丰富度、灾难频率）

### 输出
- 地形网格（山脉、平原、水域、洞穴、海岸）
- 气候区划（温度、湿度、季节参数）
- 水文网络（河流、湖泊、地下水）
- 初始物种布置（每个个体的物种、位置、基因、年龄、性别）
- 初始植被分布

### 实现优先级：🔴 必须（没有世界就没有游戏）

### 原型阶段最小版本
- 一张固定尺寸的 2D 网格地图
- 3-5 种地形类型
- 1 条河流
- 2 个猿类小群体（共 20-30 个体）
- 2-3 种其他动物物种
- 简单的植被分布（果树密度 + 草地覆盖）

---

## 2. 环境系统

> 持续运行，管理世界的非生物要素。

### 子系统

| 子系统 | 职责 | 更新频率 |
|--------|------|----------|
| 昼夜周期 | 光照变化、活动时段切换 | 每 tick |
| 天气 | 降雨、干旱、风暴、温度波动 | 每天 |
| 季节 | 食物产出系数、动物行为模式切换 | 每季 |
| 气候变化 | 长期趋势（渐暖/渐冷、干湿交替） | 每百年 |
| 地质事件 | 火山、地震、河流改道（极低频） | 随机触发 |

### 输出（供其他系统读取）
- 当前区域温度、湿度、光照
- 食物可用性系数（影响植被产出）
- 灾难事件标记
- 水位变化

### 实现优先级：🔴 必须

### 原型阶段最小版本
- 简化为：旱季/雨季两季交替
- 河流水位随季节变化
- 一个可触发的旱灾事件（用于 MVP 场景）
- 无需地质系统

---

## 3. 生态系统

> 管理植被生长、食物链、种群层面的承载力。

### 职责
- 植被生长与再生（受环境系统的温度/水分驱动）
- 食物分布的空间计算
- 环境承载力的动态评估（某区域能养活多少个体）
- 腐烂与分解（尸体处理）

### 关键数据
- 每个地格的食物类型与数量
- 每个地格的承载力上限
- 食物的季节性产出曲线

### 实现优先级：🔴 必须

### 原型阶段最小版本
- 果树按区域分布，有季节性产出
- 食物被消耗后按固定速率再生
- 尸体在固定时间后消失（简化分解）
- 不需要完整食物网——只需"植物 → 猎物 → 捕食者"的三级链

---

## 4. 个体系统（核心系统）

> 模拟每一个生物个体。这是整个游戏最核心的系统。

### 每个个体持有的数据

```
个体
├── 身份：物种、年龄、性别、唯一ID
├── 身体：体型、各部位健康状态、饥饿度、疲劳度
├── 位置：当前坐标、移动目标
├── 性格：勇敢/谨慎、好奇/保守、社交/独立、攻击/温顺
├── 情绪：当前情绪状态（平静/压力/哀伤/恐惧/兴奋）
├── 记忆：地点记忆、个体记忆、事件记忆、发现记忆
├── 技能：已知行为列表 + 各行为熟练度
├── 社交：关系列表（对象ID + 关系类型 + 强度）
├── 基因：遗传属性（影响后代的性格基线与身体特征）
└── 生命阶段：幼年 / 青少年 / 成年 / 老年
```

### 行为决策（每 tick）

```
感知环境 → 评估需求（饥饿？危险？社交？好奇？）
         → 参考性格权重 + 记忆 + 情绪
         → 选择行为（觅食/逃跑/社交/探索/休息/...）
         → 执行行为
         → 更新状态（饱腹度/位置/记忆/情绪/社交关系）
```

### 实现优先级：🔴 必须（没有个体就没有世界）

### 原型阶段最小版本

**猿类个体（完整版）：**
- 基本需求：饥饿、疲劳
- 基本行为：觅食、移动、休息、逃跑
- 简化性格：2 轴（勇敢/谨慎 + 好奇/保守）
- 简化记忆：记住食物位置、记住危险位置
- 简化社交：知道谁是亲属、基本的亲近/回避
- 简化情绪：平静 / 压力 两种状态
- 生命周期：出生 → 成长 → 成年 → 衰老 → 死亡
- 繁殖：成年后按条件配对产生后代

**其他动物个体（简化版，初期）：**
- 基本需求 + 基本行为
- 简单领地/活动范围
- 繁殖与死亡
- 对猿类个体的反应（捕食/逃避/无视）

## 5. 社会系统

> 处理个体之间的社交关系与群体结构。

### 职责
- 维护个体间的关系图（亲子、配偶、同伴、敌对）
- 计算群体边界（基于个体间互动频率的聚类）
- 处理等级关系（支配与服从）
- 触发社交行为（理毛、玩耍、分享食物、威胁、安抚）
- 检测群体分裂条件

### 输出
- 群体归属（每个个体属于哪个群体/独居）
- 群体内等级序列
- 关系网络变化事件（供编年史记录）

### 实现优先级：🟡 重要（社交让个体"活起来"）

### 原型阶段最小版本
- 亲子关系（自动建立）
- 基于空间邻近的简单群体判定
- 个体间的亲近度（通过共处时间累积）
- 不需要完整的等级系统——只需要"谁是群体中心"的简单计算

---

## 6. 演化系统

> 管理长时间尺度上的身体与行为变化。

### 子系统

| 子系统 | 时间尺度 | 驱动力 |
|--------|----------|--------|
| 行为演化 | 代际 | 观察学习 + 偶然发现 + 练习 |
| 技术/工具 | 数十代 | 偶然事件 + 能力积累 + 材料接触 |
| 身体演化 | 数百代 | 自然选择（基因 + 环境压力） |
| 认知演化 | 数百代 | 社会复杂度 + 营养 + 环境挑战 |

### 关键机制
- **偶然事件触发器**：每 tick 有极低概率触发"偶然事件"（石头碎裂、雷击、食物接触火源等），概率受个体性格（好奇心）和技能积累影响。
- **技能传播**：熟练个体的行为可被观察学习；依赖空间邻近性和观察者的能力。
- **遗传变异**：后代的基因在亲代基础上有小幅随机变异；环境选择压力决定哪些变异被保留。

### 实现优先级：🟡 重要（但可以分阶段实现）

### 原型阶段最小版本
- **只做行为演化**：个体通过重复实践提高技能熟练度，幼崽通过观察学习新行为。
- 不需要身体演化（太慢，原型时间尺度内看不到）。
- 不需要完整的偶然事件系统——手动放入一两个可触发的关键事件即可。
- 一个最简单的"工具使用"行为：猿可以捡起石头并用于敲击。

---

## 7. 文化系统

> 管理行为/知识在个体间和代际间的传递。

### 职责
- 追踪每项行为/技能的"发明者"和传播链
- 管理"谁从谁那里学到了什么"
- 检测文化断裂（最后一个掌握者死亡 → 技能失传）
- 记录文化里程碑

### 实现优先级：🟢 可后置（原型阶段的简化版见演化系统）

### 原型阶段最小版本
- 与演化系统合并：追踪"谁知道某项技能"即可
- 文化断裂检测：当无个体掌握某技能时生成事件

---

## 8. 伤害系统

> 管理个体受伤、康复、永久伤残。

### 职责
- 根据事件（攻击、跌落、极端温度）在特定身体部位产生伤害
- 计算伤害对功能的影响（跛行、单眼、残肢）
- 管理康复过程
- 标记永久伤残

### 实现优先级：🟢 可后置（原型阶段用简化版）

### 原型阶段最小版本
- 简单的 HP 系统 + "受伤/健康"二态
- 受伤降低移动速度和觅食效率
- 严重受伤可致死
- 不需要身体部位级精度

---

## 9. 观察系统（玩家界面）

> 玩家与世界的接口。不影响模拟。

### 子系统

| 子系统 | 职责 |
|--------|------|
| 视角控制 | 移动、缩放、跟随个体 |
| 尺度切换 | 宏观/中观/微观 |
| 信息层叠加 | 生态/资源/社会/行为层 |
| 个体档案 | 查看个体详细信息 |
| 标记与命名 | 玩家自定义标签 |
| 通知系统 | 关键事件提醒（可配置） |
| 时间控制 | 加速/减速/暂停 |
| 编年史 | 事件浏览、个体传记、血统树 |
| 存档管理 | 保存/加载/分支 |

### 实现优先级：🔴 必须（没有观察就没有游戏）

### 原型阶段最小版本
- 2D 俯视图 + 鼠标拖拽移动
- 点击个体查看基本信息（名字、年龄、饥饿度、关系）
- 时间控制（暂停/1x/2x/4x）
- 点击个体后可命名
- 简单的事件日志（文字列表）
- 基本存档/加载

---

## 10. 干预系统

> 玩家偶发地影响模拟。这是最微妙的系统。

### 设计原则（来自 04）
- 干预不是主动控制，而是被世界触发的选项
- 干预有不可预测的代价
- 不干预也是选择

### 实现优先级：🟡 重要（核心体验的一部分，但可简化）

### 原型阶段最小版本
- **一种干预类型**：降雨（改变局部水资源）
- **触发条件**：当旱灾达到阈值时出现选项
- **效果**：改变河流水量分布
- **代价**：自然产生——上游增水 = 下游可能减水
- 不需要复杂的"干预点数"或"使用次数"系统——MVP 只需一次选择

## 11. 系统间耦合关系

```
环境 ──→ 生态（温度/水分影响食物产出）
环境 ──→ 个体（温度影响活动、灾难直接伤害）
生态 ──→ 个体（食物可用性驱动觅食行为）
个体 ──→ 个体（社交互动、冲突、繁殖）
个体 ──→ 生态（消耗食物、杀死猎物、改变环境）
个体 ──→ 社会（互动频率产生群体边界与等级）
社会 ──→ 个体（群体归属影响行为决策）
个体 ──→ 演化（行为积累 + 偶然事件 → 新能力）
演化 ──→ 个体（新行为加入个体的可用行为集）
演化 ──→ 文化（新行为可被传播）
文化 ──→ 个体（学习到的行为影响决策）
干预 ──→ 环境（玩家的干预作用于环境子系统）
```

### 关键原则
- **单向信息流**：玩家界面层只读取模拟数据，不回写（除干预系统外）。
- **个体是中心节点**：几乎所有系统都通过个体进行耦合。个体是模拟的原子单元。
- **环境是顶层驱动力**：大部分变化的根源在环境变化，向下驱动生态→个体→社会。

## 12. 开发优先级总览

### 阶段一：最小可玩（Playable Minimum）
> 目标：能跑起来、能看、能产生最基本的"我想继续看"的感觉。

| 系统 | 需要做到 |
|------|----------|
| 世界生成 | 固定小地图 + 手动布置 |
| 环境 | 旱季/雨季切换 + 旱灾事件 |
| 生态 | 果树产出 + 简单食物链 |
| 个体 | 基本需求 + 基本行为 + 简化性格 + 生命周期 |
| 社会 | 亲子关系 + 群体判定 |
| 观察 | 2D视角 + 个体信息 + 时间控制 + 命名 |
| 干预 | 一次降雨选择 |

### 阶段二：情感产生（Emotional Minimum）
> 目标：玩家开始"在乎"某只猿。

| 系统 | 新增 |
|------|------|
| 个体 | 完整性格系统 + 记忆 + 情绪 |
| 社会 | 理毛/冲突/安抚等可见社交行为 |
| 演化 | 技能熟练度 + 观察学习 |
| 观察 | 关系网络可视化 + 个体档案 |
| 编年史 | 基本事件日志 + 个体传记 |

### 阶段三：涌现发生（Emergence Minimum）
> 目标：群体开始产生"像文化"的东西。

| 系统 | 新增 |
|------|------|
| 演化 | 偶然事件触发 + 工具使用行为链 |
| 文化 | 行为传播追踪 + 文化断裂检测 |
| 伤害 | 身体部位级伤害 + 永久伤残 |
| 个体 | 材料认知 + 工具制造 |
| 观察 | 信息层叠加 + 行为传承图 |

### 阶段四：完整循环（Full Loop）
> 目标：三层核心循环全部运转。

| 系统 | 新增 |
|------|------|
| 世界生成 | 种子驱动 + 参数配置 |
| 环境 | 完整天气/季节/气候/地质 |
| 生态 | 完整食物网 + 承载力 |
| 个体 | 完整非猿类模拟 |
| 干预 | 完整触发/代价/后果系统 |
| 观察 | 回溯分支 + 树状视图 |

## 附录：系统数据字典（原型阶段）

### 个体（Entity）
```
id:          uint        // 唯一标识
species:     enum        // 物种
age:         float       // 年龄（游戏年）
sex:         enum        // 性别
position:    (x, y)      // 位置
hunger:      float       // 饱腹度 [0, 1]
fatigue:     float       // 疲劳度 [0, 1]
health:      float       // 健康值 [0, 1]
mood:        enum        // 情绪（calm / stressed）
courage:     float       // 勇敢-谨慎 [-1, 1]
curiosity:   float       // 好奇-保守 [-1, 1]
skills:      map<行为, 熟练度>
relations:   list<(id, type, strength)>
memories:    list<Memory>
alive:       bool
parent_ids:  (id, id)    // 双亲
name:        string?     // 玩家命名（可空）
```

### 记忆（Memory）
```
type:        enum        // location / entity / event / discovery
target:      any         // 相关位置/个体/事件
valence:     float       // 正面/负面 [-1, 1]
strength:    float       // 强度 [0, 1]（随时间衰减）
timestamp:   float       // 记忆形成时间
```

### 世界事件（WorldEvent）
```
type:        enum        // birth / death / discovery / split / extinction / ...
timestamp:   float
location:    (x, y)
involved:    list<id>    // 相关个体
description: string      // 客观描述
```
