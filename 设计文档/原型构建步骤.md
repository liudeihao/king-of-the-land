# 原型构建步骤（Unity 2D）

> 本文档是"里程碑二：最小技术原型"的逐步执行指南。
> 原则：**只做三件事** —— 个体、食物压力、观察。其他一概不碰。

---

## 前置准备

1. 安装 Unity Hub → 安装 Unity 6 LTS（或任何你能装的最新 LTS 版本）
2. Unity Hub → New Project → 选 **2D (URP)** 模板
3. 项目名：`LandKing_Prototype`，路径随意
4. 删掉默认场景里的多余物体，只留 Main Camera

---

## 第一步：20×20 网格地图

### 目标
> 屏幕上出现一张 20×20 的彩色方块地图。绿色=草地，蓝色=河流，深绿=果树。

### 做法

1. 创建脚本 `MapGenerator.cs`，挂到一个空 GameObject 上
2. 用一个 `int[20,20]` 数组表示地图，手动定义：
   - 中间一列竖着的 = 河流（值=1）
   - 河流两侧各随机撒 8-10 个 = 果树（值=2）
   - 其余 = 草地（值=0）
3. 用 `SpriteRenderer` 或 `GameObject.CreatePrimitive(Quad)` 逐格渲染
   - 草地 → 浅绿色
   - 河流 → 蓝色
   - 果树 → 深绿色
4. 调整 Camera 的 `orthographicSize` 让整张地图刚好全屏可见

### 数据结构

```csharp
public enum TileType { Grass, River, FruitTree }

public class MapData
{
    public TileType[,] tiles = new TileType[20, 20];
}
```

### 完成标准
- 运行后看到 20×20 的彩色网格
- 河流在中间，两侧有果树
- **耗时上限：2 小时**

---

## 第二步：放置猿类个体

### 目标
> 地图上出现 10 个小圆点（猿），它们站在随机的草地/果树格子上。

### 做法

1. 创建脚本 `Ape.cs`，作为猿的组件
2. 创建一个 prefab：一个小圆形 Sprite（或用 `SpriteRenderer` + 内置圆形），颜色棕色
3. 创建脚本 `WorldManager.cs`，在 `Start()` 中：
   - 在河流左侧随机选 5 个非河流格子，实例化 5 只猿（Group A）
   - 在河流右侧随机选 5 个非河流格子，实例化 5 只猿（Group B）

### Ape 数据结构（极简版，只这些字段）

```csharp
public class Ape : MonoBehaviour
{
    public int id;
    public float hunger = 1f;      // 1=饱，0=饿死
    public float health = 1f;      // 1=健康，0=死亡
    public float age = 0f;         // 游戏年
    public bool alive = true;
    public string nickname = "";   // 玩家命名，初始为空
    public Vector2Int gridPos;     // 当前所在格子
}
```

### 完成标准
- 运行后地图上出现 10 个棕色圆点，分布在河流两侧
- 点击控制台能打印每只猿的 hunger 值
- **耗时上限：1 小时**

---

## 第三步：时间系统 + 饥饿

### 目标
> 时间在流逝，猿会饿。饿到 0 就死（从地图上消失）。

### 做法

1. 创建脚本 `TimeManager.cs`（挂到空物体上）
   - `float tickInterval = 1f;` — 每秒一个 tick
   - `int tickCount = 0;` — 总 tick 数
   - `float timeScale = 1f;` — 时间倍速（1x/2x/4x）
   - 每个 tick 广播事件（或直接调用所有 Ape 的 `OnTick()`）
2. `Ape.OnTick()`：
   - `hunger -= 0.02f;` （50 tick ≈ 50 秒后饿死，如果不吃东西）
   - 如果 `hunger <= 0` → `alive = false`，禁用 GameObject 或变灰
3. UI：左上角显示当前 tick 数 + 时间倍速
4. 键盘控制倍速：
   - `1` 键 → 1x
   - `2` 键 → 2x
   - `3` 键 → 4x
   - `Space` → 暂停/继续

### 完成标准
- 运行后什么都不做，大约 50 秒后猿开始死亡
- 按 `2` 加速后死亡来得更快
- 按空格能暂停
- **耗时上限：2 小时**

---

## 第四步：觅食行为

### 目标
> 猿会自动走向最近的果树，到了之后吃东西恢复饥饿值。果树有有限的食物量。

### 做法

1. 给 `MapData` 加字段：
   ```csharp
   public float[,] food = new float[20, 20]; // 果树格子初始 food=5.0，其他=0
   ```
2. 果树食物再生：每 10 个 tick，所有果树格子 `food += 0.5`，上限 5.0
3. `Ape.OnTick()` 行为决策（极简状态机）：
   ```
   if hunger < 0.7:
       找到最近的有食物的果树格子
       每 tick 向该格子移动一步（上下左右，不能走河流）
       到达后：hunger += 0.3, food[x,y] -= 0.3
   else:
       随机游荡（随机方向移动一步，或原地不动）
   ```
4. 移动：每 tick 移动一格（简单的像素跳跃，不需要平滑动画，但如果想加 `Vector2.MoveTowards` 也行）
5. 不能走进河流格子（简单碰撞：目标是河流就换方向）

### 完成标准
- 猿会在饥饿时自动走向果树
- 到达果树后饥饿值恢复
- 如果所有果树食物耗尽，猿开始饿死
- 10 只猿在地图上来回走动看起来"活着"
- **耗时上限：3 小时**

## 第五步：点击查看 + 命名

### 目标
> 点击一只猿，显示它的信息。可以给它起名字。

### 做法

1. 给每只猿的 GameObject 加 `BoxCollider2D`
2. 创建脚本 `SelectionManager.cs`：
   - `Update()` 中检测鼠标点击 → `Physics2D.Raycast` → 命中猿 → 设为"选中"
   - 选中的猿周围画个白色圈（子物体 SpriteRenderer，默认隐藏，选中时显示）
3. 创建一个简单的信息面板（Canvas → Panel），显示：
   ```
   ID: 7
   名字: [点击命名]
   饥饿: ████░░ 68%
   健康: ██████ 100%
   年龄: 3.2 年
   状态: 觅食中
   ```
4. 名字区域可点击 → 弹出 InputField → 输入后保存到 `ape.nickname`
5. 命名后，猿头顶显示名字文字（`TextMeshPro` 子物体）

### 完成标准
- 点击猿弹出信息面板
- 能输入名字，名字显示在猿头顶
- 信息面板实时更新（饥饿值随 tick 变化）
- **耗时上限：3 小时**

## 第六步：旱灾 + 一次干预

### 目标
> 运行一段时间后河流水位下降，果树食物产出减少，猿开始挨饿。玩家获得一次"降雨"按钮。

### 做法

1. `MapData` 加字段：`float waterLevel = 1.0f;`
2. `TimeManager` 在第 100 个 tick 后开始旱灾：
   - 每 tick：`waterLevel -= 0.002f`，下限 0
   - 河流两侧果树的食物再生速率 = `基础速率 × waterLevel`
   - 当 `waterLevel < 0.3` 时，部分果树格子变成棕黄色（视觉反馈：旱了）
3. 当 `waterLevel < 0.2` 时，屏幕上方出现按钮：**"降雨"**
   - 点击后：`waterLevel = 0.8`
   - **但只对河流左侧的果树生效**（右侧的 waterLevel 继续下降）
   - 按钮消失，不可再用
4. 不点击也可以 → 旱灾持续，所有猿面临饥饿

### 完成标准
- 旱灾开始后果树变色、食物减少、猿开始挨饿
- 点击"降雨"后左侧恢复，右侧继续恶化
- 如果右侧有你命名的猿 → 你会心痛
- **耗时上限：2 小时**

---

## 第七步：简单事件日志

### 目标
> 屏幕右侧有一个滚动文字列表，记录世界发生的事。

### 做法

1. Canvas 右侧放一个 `ScrollView` + `TextMeshProUGUI`
2. 创建 `EventLog.cs`（单例），提供 `Log(string message)` 方法
3. 在以下时机写日志：
   - 猿死亡：`"[名字/ID] 因饥饿死亡 (tick 234)"`
   - 旱灾开始：`"水位开始下降 (tick 100)"`
   - 玩家降雨：`"降雨发生在西侧 (tick 187)"`
   - 果树枯竭：`"区域(3,5)的果树食物耗尽"`
4. 日志列表自动滚动到底部，最多显示最近 50 条

### 完成标准
- 能在日志中看到猿的死亡记录
- 旱灾和降雨事件被记录
- 回看日志能拼凑出"发生了什么"
- **耗时上限：1.5 小时**

## 核心脚本清单（只需要这些文件）

```
Scripts/
├── MapGenerator.cs      // 生成并渲染 20×20 地图
├── MapData.cs           // 地图数据（格子类型、食物量、水位）
├── Ape.cs               // 猿的数据 + 行为（觅食/移动/死亡）
├── WorldManager.cs      // 初始化世界、生成猿、管理全局状态
├── TimeManager.cs       // tick 推进、倍速控制、旱灾触发
├── SelectionManager.cs  // 鼠标点击选择猿
├── UIManager.cs         // 信息面板 + 命名 + 降雨按钮
└── EventLog.cs          // 事件日志
```

**8 个脚本。不多。不少。够了。**

- ❌ 性格系统
- ❌ 记忆系统
- ❌ 情绪系统
- ❌ 社交关系图
- ❌ 等级结构
- ❌ 技能传播 / 文化系统
- ❌ 身体部位伤害
- ❌ 繁殖系统
- ❌ 多物种（只有猿）
- ❌ 气候长期趋势
- ❌ 地质事件
- ❌ 回溯/存档
- ❌ 编年史浏览器
- ❌ 信息层叠加
- ❌ 美术资产
- ❌ 音效

## 完成后你会得到什么

一个画面极其简陋的原型：
- 20×20 彩色方块
- 10 个棕色圆点在走来走去
- 右侧一行行文字在滚动
- 你点了其中一个，给它起了个名字
- 旱灾来了，它开始饿
- 你按了降雨，它活了
- 但你突然注意到右边那几个不动的圆点——它们已经灰了
- 你打开日志，发现"圆点7 因饥饿死亡 (tick 203)"

如果这一刻你感到了什么——

**核心循环成立。继续推进。**

如果什么都没感到——

**回来改设计，不要往前冲。**
